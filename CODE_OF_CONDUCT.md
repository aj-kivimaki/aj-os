# Code of Conduct

AJ-OS is a small project with a single maintainer, developed in the open. This is short because it
can afford to be.

## The standard

**Be decent.** Assume good faith, disagree about the work rather than the person, and accept that
the maintainer may say no to something you think is obviously right.

Not welcome: harassment, personal attacks, or making anyone feel unwelcome for who they are.

## Scope

Issues, pull requests, discussions, and any other space where this project is the subject.

## Enforcement

Report to the maintainer via
**[GitHub Security Advisories](https://github.com/aj-kivimaki/aj-os/security/advisories/new)** if it
is sensitive, or by opening an issue if it is not.

**What to expect:** one person reading it, in their own time, and acting as they judge appropriate —
up to blocking. There is **no committee, no appeal process, and no response-time commitment.**
Saying otherwise would be inventing a process that does not exist.
