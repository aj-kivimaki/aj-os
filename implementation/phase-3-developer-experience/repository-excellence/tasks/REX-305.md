# Purpose

CONTRIBUTING's architecture rule describes **3 of 11** top-level `src/` modules. Eight have no slot,
including the live agent layer (`src/agent/`, `src/api/`, `src/handbook/`) that F-022 flagged in M1
and M1 deferred here. A reader given the rule cannot place most of the tree. This task gives every
module a documented home and expected lifetime, using the taxonomy REX-D1 established.

---

# Objective

CONTRIBUTING names every top-level `src/` module with its role and expected lifetime; the README
subsystem table includes the agent layer. No architecture document is *amended* — REX documents at
README/CONTRIBUTING level and **recommends** an ADR where a mismatch with ARCH-001 is exposed
(REX-D1's authority ordering).

---

# Scope

## Included
- Extend CONTRIBUTING's architecture section to cover all 11 modules: `agent`, `api`, `cli`, `config`,
  `context-builder`, `end-of-session`, `handbook`, `ingestion`, `knowledge`, `platform`, `products`
  (F-023).
- Add the agent layer (`src/agent/`, `src/api/`) to the README subsystem table (F-022 doc half).
- State each module's **expected lifetime** per REX-D1's taxonomy (permanent platform / product /
  transitional / to-be-migrated).

## Not Included
- **Amending ARCH-001 or any architecture document** — REX-D1: document + recommend, never author an
  ADR. Where the agent layer's home genuinely conflicts with ARCH-001, **recommend** an ADR.
- The file-naming rule — **REX-304**.
- Resolving the two config systems or the agent layer's testing gap — those are **M4** (F-054, F-055).

# Implementation Notes

**Describe what is, and mark what is transitional.** The agent layer is live but MCP-migration is
planned (per the memory of record and ADR direction); state it as present with its expected lifetime,
not as permanent architecture. That is REX-D1's taxonomy doing its job.

**Recommend, do not decide.** If placing a module truthfully requires an architectural decision
ARCH-001 does not carry, the output is a **recommendation for an ADR**, recorded — not an amendment.
This is the authority ordering the reviewer required at M1.

**The taxonomy must be falsifiable.** *"Every top-level `src/` module appears in CONTRIBUTING"* is a
checkable claim — a test or a documented `grep` over `ls src/` vs the doc. Prefer the mechanical
check to a prose assertion (the M1 assertion-inventory lesson).
