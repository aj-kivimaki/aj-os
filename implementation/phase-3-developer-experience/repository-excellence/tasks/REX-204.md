# REX-204 — Formatting

> **Status:** Planned

> **Protected outcome:** **mechanical — and provably so**

> **Implementation Package:** REX — Repository Excellence Review · **Milestone:** M2

---

# Purpose

**Why does this task exist?**

No formatter exists. The code is formatted consistently **by hand** — 2-space, double quotes,
semicolons, ~90 columns. That has worked because one person wrote it. It is not enforced, it is not
checkable, and it silently becomes review noise the moment a second contributor appears.

**This task is also the milestone's proof.** [REX-D10](../decisions/REX-D10.md) rests on it: a
formatter is **deterministic**, so `format(pre-M2 tree) == post-M2 tree` demonstrates objectively
that the formatting commit contains **no semantic edit**. Every other "mechanical only" claim in M2
is a reviewed judgement; this one is a computation.

---

# Objective

Introduce Biome's formatter and `.editorconfig`, in a diff proven to be exactly the formatter's work
and nothing else.

---

# Scope

## Included
- Biome formatter config ([REX-D7](../decisions/REX-D7.md)); `.editorconfig`; `format` / `format:check`
  scripts; the first-run diff as an **isolated commit**; `.git-blame-ignore-revs`.

## Not Included
- **Linting** — REX-205. Landing them together makes both diffs unreviewable.
- Any semantic change whatsoever. **The proof below forbids it by construction.**
- Changing the existing style. Configure Biome to match what the repository already does — **this
  task is not a style decision**, and re-styling 12k lines would bury the one thing worth reviewing.

# Implementation Notes

**Configure to match, then run.** Read the existing style; set Biome to reproduce it; run it. If the
first-run diff is large, the config is wrong — **a formatter matching the repository's existing hand
style should barely change anything.** A big diff means we chose a new style, which is out of scope.

**One commit, nothing else in it.** `.git-blame-ignore-revs` so `git blame` survives.

**The proof is the deliverable.** Check out the pre-M2 tree, run the formatter, diff against the
post-M2 tree: **they must be identical.** Then verify the proof can fail — introduce a semantic edit
into the formatted tree and confirm the check reports it. *A proof that cannot fail is not evidence.*
