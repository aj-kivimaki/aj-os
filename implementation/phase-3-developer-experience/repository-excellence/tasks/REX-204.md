# Purpose

**Why does this task exist?**

No formatter exists. The code is formatted consistently **by hand** — 2-space, double quotes,
semicolons, ~90 columns. That has worked because one person wrote it. It is not enforced, it is not
checkable, and it silently becomes review noise the moment a second contributor appears.

**This task is also the milestone's proof.** [REX-D10](../decisions/REX-D10.md) rests on it: a
formatter is **deterministic**, so `format(pre-M2 tree) == post-M2 tree` demonstrates objectively
that the formatting commit contains **no semantic edit**. Every other "mechanical only" claim in M2
is a reviewed judgement; this one is a computation.

---

# Objective

Introduce Biome's formatter and `.editorconfig`, in a diff proven to be exactly the formatter's work
and nothing else.

# Implementation Notes

**Configure to match, then run.** Read the existing style; set Biome to reproduce it; run it. If the
first-run diff is large, the config is wrong — **a formatter matching the repository's existing hand
style should barely change anything.** A big diff means we chose a new style, which is out of scope.

**One commit, nothing else in it.** `.git-blame-ignore-revs` so `git blame` survives.

**The proof is the deliverable.** Check out the pre-M2 tree, run the formatter, diff against the
post-M2 tree: **they must be identical.** Then verify the proof can fail — introduce a semantic edit
into the formatted tree and confirm the check reports it. *A proof that cannot fail is not evidence.*

# Worklog Summary

_2026-07-17._

Biome formatter + `.editorconfig` + `format`/`format:check` scripts; `npm run ci` now gates format
first. **104 files, 504 insertions, 851 deletions — all within `src/` and `tests/`.** Isolated
commit; SHA recorded in `.git-blame-ignore-revs`.

**The proof.** Take the pre-REX-204 tree, run the formatter on it independently, compare:
**identical across all 104 changed files.** Then **verified able to fail** — smuggling a semantic
edit *and formatting it* so it looks like formatter output is the hardest case, and the proof catches
it.

**`lineWidth: 90`, measured rather than assumed.** At 80: ~1058 lines rewritten, and it splits
imports the author deliberately kept whole. At 90: ~504, and `createEndOfSessionWorkflow.ts` is
byte-identical. `src` p99 = 91. **The task's own rule decided it** — *"a big diff means we chose a
new style"* — and 80 would impose a style narrower than this code has ever used.

## Two traps, both found by running it

**1. The formatter rewrote the frozen archive.** ~200 files of preserved v1 history —
`docs/archive/README.md`: *"not maintained or updated."* Reverted; excluded with explicit negations.

**2. The root cause: `biome.json` cannot contain comments.** Biome reported parse errors,
**silently fell back to defaults, and formatted everything anyway** — archive included, at width 80.
`--write` ran regardless. Moved to **`biome.jsonc`**; verified zero parse errors and that the
exclusions then take effect.

---

# Lessons Learned

- **A config that fails to parse does not stop the run — it changes it.** Biome printed parse errors
  *above* its output and formatted the entire repository with defaults, including ~200 files of
  frozen archive. The blast radius was everything; the signal was a warning nobody had to read.
  **`--write` should refuse to run on an unparseable config**, and since it doesn't, the config must
  be proven to take effect rather than assumed to.

- **F-029's premise is false, and its falseness is the argument.** The finding says the code is
  *"formatted consistently by hand."* **It isn't** — `src` and `tests` were written to different
  widths, and some lines were split at ~80 while others ran to ~90. **No width leaves it untouched**,
  so the task's *"a formatter matching the existing style should barely change anything"* rule could
  not be satisfied literally. The inconsistency is precisely why a formatter is needed.

- **I reverted my own work twice with `git checkout -- .`** while cleaning up the archive incident,
  silently losing the `format`/`format:check` scripts — and `npm run ci` then reported **green while
  no longer running the format gate at all.** A green pipeline that has quietly stopped checking
  something is worse than a red one. Caught by running `format:check` standalone rather than trusting
  the aggregate.

- **The proof is the deliverable, not the formatting.** The 504 lines are unremarkable. What REX-204
  actually produced is a **reusable, deterministic demonstration that a mechanical change is
  mechanical** — the only claim in M2 that is a computation rather than a reviewed judgement.
