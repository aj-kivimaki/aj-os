# REX-208 — Coverage: Measured, Not Gated

> **Status:** Planned
>
> **Protected outcome:** **governance — measurement**
>
> **Implementation Package:** REX — Repository Excellence Review · **Milestone:** M2

---

# Purpose

**Why does this task exist?**

**Coverage is not measured at all.** `@vitest/coverage-v8` is not installed. `vitest.config.ts:14`
says *"No coverage/reporters/timeouts tuning yet."* `coverage/` is **gitignored in anticipation of a
thing that never happens.**

So the repository cannot answer a basic question — *what is actually tested?* — and the inventory
already knows the answer is uneven: `KnowledgeAssistant.ts` is **410 lines with zero tests** (F-053);
the entire agent layer is untested (F-054). **Those were found by reading, not by measuring.**
Measurement would have found them, and will find the ones reading missed.

---

# Objective

**Measure coverage. Report it. Do not gate on it.**

---

# Scope

## Included
- `@vitest/coverage-v8`; coverage config in `vitest.config.ts`; `test:coverage` script; coverage
  **reported in CI**.
- The **baseline number**, recorded as a finding-adjacent fact.

## Not Included
- ⚠️ **Thresholds.** Deliberate, and the reason this task exists separately from REX-201. **A
  threshold picked before measuring is a number, not a standard** — it would either be trivially met
  (worthless) or arbitrary (gamed). **Gating is a later review's, on this baseline.**
- Writing tests to raise coverage — **M4** owns `KnowledgeAssistant` and the agent layer.
- Failing CI on a coverage drop.

---

# Dependencies

## Requires
- **REX-201** — coverage is reported by the pipeline.
- **REX-203** — measuring before the suite type-checks would baseline a suite about to change.

## Enables
- A future gating decision, **on evidence**.
- M4's DI work — which will move the number, and now visibly.

# Implementation Notes

**Report, don't gate. This is the whole discipline of the task.** The temptation is to add a
threshold "while we're here" — and the plan already ruled it out for a reason. **Measure first;
decide later, on evidence.** That is the same restraint REX applied to its own Measurable-vs-Judgement
idea.

**Record the baseline as a fact, not a target.** And **do not put the number in a document** — that is
F-010/F-011 all over again, the hard-coded metric that drifts by construction. **The number lives in
CI output, where it regenerates.** M1 spent two findings learning this.

**Expect the number to be unflattering, and let it be.** Two known holes are large. **A coverage
report that flatters is not measuring.**

**Exclude what should not count** — `dist/`, `archive/`, config files, `tests/` itself. Do **not**
exclude untested production code to make the number look better; **that is the one exclusion that
would defeat the task.**

# Validation

- **No threshold configured** — verified by reading the config.

  the exclusions are wrong* — the report must be able to show bad news.
- **No number written into any document.**
