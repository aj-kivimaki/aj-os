# Implementation Notes

**Characterization tests FIRST.** Before touching any guard, confirm the existing store suites cover
symlink escape, NUL, `..`, absolute paths, and canonical-rejection, and are **green on current
behaviour**. If a case is missing, add it and prove it green **before** the refactor — the frozen
validation strategy. A guard refactor with no failing-case coverage is not safe to make.

**F-050 is a defect repair, not a rewrite.** Adopt the `realpathIfExists` shape; do not redesign the
guard. The WikiStore must still reject every escape it rejects today — proven by the suite.

**F-051 is the only consolidation, and it is in-module.** One private helper inside `context-builder`;
five call sites updated. No new cross-module import. `immutable.ts` stays as-is (EOS-005).

**Restore-by-rewrite, never `git checkout`, on uncommitted work** (the standing M3-A/M3-B lesson).
