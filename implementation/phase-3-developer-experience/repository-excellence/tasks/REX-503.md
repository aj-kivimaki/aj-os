# Purpose

`wiki.ts`'s `printReport` `console.log`s directly and is untestable, while `session.ts`'s
`formatSessionReport` returns `string[]` **specifically so it can be tested** — same layer, two
conventions, and one is the exemplar. Meanwhile seven test helpers are duplicated across suites against
a documented per-suite-inlining convention. This task makes the report testable and applies
[REX-D4](../decisions/REX-D4.md) to the helpers.

---

# Objective

`printReport` follows `session.ts`'s exemplar (format → return, print separately), gaining a test; the
duplicated helpers are consolidated or reaffirmed per REX-D4.

---

# Scope

## Included
- **F-065** — refactor `wiki.ts`'s report printing to a pure `format*` that returns `string[]` (like
  `formatSessionReport`), with a thin printer around it; add a test for the formatter.
- **F-064** — per the REX-D4 ruling: consolidate the byte-identical cross-cutting infrastructure stubs
  (`stubGenerator`, `makeProvider`, `stubGitPort`) into shared support **only where the criteria hold**;
  reaffirm per-suite contract fixtures (`candidate`, `item`) as a recorded result.

## Not Included
- Changing what the report **says** or what `wiki build` **does** — F-065 is a testability refactor.
- Coupling contract fixtures across suites (the `support.ts` convention holds).

# Implementation Notes

**Copy `session.ts`'s exemplar exactly.** `formatSessionReport` returns `string[]` and documents *why*
(testable without stdout capture); `wiki.ts`'s formatter should mirror it, with the `console.log` in a
one-line printer the test does not need to touch. **Same layer, now one convention.**

**Consolidate a stub only after proving it byte-identical** (REX-D4). If a `stubGenerator` differs per
suite, it stays inline and the difference is the recorded reason — a "keep" is a valid result.

**No behavioural change.** The report's content is identical; only its testability changes.
