# Purpose

**Why does this task exist?**

Two documents assert a capability that a **frozen decision deliberately excludes**, and a live
subsystem appears in **no architecture document at all**. These are the two places where AJ-OS's
documentation misrepresents not a status, but the platform's **shape**.

**The contradiction.** `docs/README.md:74` says *"SPEC-003 | End-of-Session (**owns commits**)"* and
`ROADMAP.md:26` says the workflow *"**owns git commits** (the engine never commits)"*. SPEC-003 v1
**performs no git write** — deferred per ADR-002 (*"version control belongs to orchestration"*) and
AJS-005 §7, listed under *Not Included* and *Deferred (post-v1)*, and **verified absent at the M5
freeze by grep and at runtime**. The docs assert a capability the frozen implementation
deliberately does not have. `implementation/backlog/SPEC-003-specification-hygiene.md` item 4 even
names this fix — and it went unapplied, which is why it is here.

**The gap.** `src/agent/`, `src/handbook/`, and `src/api/` are **live** — n8n calls `/agent/ask` and
`/inbox/note` — yet they appear in no architecture document and not in README's subsystem table.
`CONTRIBUTING.md:31-38`'s "one rule" has no slot for them. The reviewer's objective for REX is
*"make the architecture accurately reflect what is actually supported today"*; **this is that task.**

# Implementation Notes

**The git-commit correction is not "delete three words."** These documents assign SPEC-003 a *role*
in the platform's story. Removing the words leaves a gap where a role was. State the **actual**
arrangement: the generator never commits (ADR-002), SPEC-003 v1 never commits either, and version
control belongs to orchestration — **which does not exist yet**. That is a real and deliberate hole
in the platform, and the docs should say so rather than paper over it by naming a component that
doesn't fill it.

**Cite the authority in the text.** Anyone who reads *"owns git commits"* today has no way to know it
is wrong. The correction should carry its own proof — ADR-002, AJS-005 §7 — so the next reader can
check rather than trust.

**On REX-D1 — respect the ceiling.** ARCH-001 is frozen and sits **above** REX in the authority
order. If the honest answer is *"the agent layer belongs in ARCH-001"*, then the honest output of
this task is a **recommendation** that an ADR be written — **not an ADR**, and not an ARCH-001 edit.
§3 permits feedback to travel upward only as a proposal. **Do not settle by invention at a lower
layer.** The recommended default is README/CONTRIBUTING plus the lifetime taxonomy, which needs no
ADR.

**State expected lifetime, not just existence.** The reviewer's taxonomy is the point: `src/api/` is
*"the current transport implementation, not automatically legacy… supported until an MCP transport
replaces it."* Documenting it as merely "existing" loses exactly the information a reader needs —
whether to build on it. **Recording that something is expected to be replaced is not the same as
recording that it is deprecated**, and the difference matters here.

**Scope guard check — the sharpest one in M1.** Correcting the contradiction describes an existing
implementation: **in scope**. Documenting the agent layer at README/CONTRIBUTING level describes what
exists: **in scope**. *Amending ARCH-001* would change what the platform's architecture claims to be:
**out of scope — defer to an ADR.** REX-D1 exists to place that line, and this task must not place it
itself.

# Lessons Learned

- **A-06 failed on the correct answer.** The assertion grepped `owns git commits` and expected no
  match; the *fix* — *"**Nobody** owns git commits"* — contains that string. **The assertion was
  matching a string, not a claim**, and a pattern derived from broken text encoded the breakage.
  Split into **A-06a** (nothing is credited) and **A-06b** (the deferral is present). **A-06b exists
  because deletion alone would have passed A-06a** — passing by saying nothing.

- **Three separate `grep` false-negatives in M1, all on correct documents.** Line wrapping
  (REX-102), blockquote markers surviving a naive join, and **BSD `sed` silently ignoring `\?`** so
  a normalisation step did nothing without erroring. Each reported a *correct* document as broken —
  the dangerous direction, since it invites "fixing" text that was already right. **M2 must
  normalise markdown with a parser, not a `sed`/`tr` pipeline**, and the warning is recorded at the
  top of the assertion inventory where M2 will find it.

- **Recommending is harder than fixing, and that is the point.** The agent layer's gap is real, its
  fix is obvious, and REX has the evidence to write the ADR. **§3 still says no.** Producing a
  recommendation strong enough to act on — while writing nothing into `docs/architecture/` — is what
  the authority order actually costs, and it is cheap compared to a quality review that quietly
  rewrites the architecture.
