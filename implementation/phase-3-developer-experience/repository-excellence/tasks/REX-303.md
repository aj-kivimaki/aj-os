# Purpose

REX-301 and REX-302 settle the public surface by **review**. Nothing stops the next PR from
re-leaking a symbol or re-adding an `export *` — because **no dead-export detector exists and
`noUnusedLocals` cannot see exports** (measured in [REX-D8](../decisions/REX-D8.md)). This task makes
M3-A's outcome **non-regressible**: the M2 ratchet, applied to the surface M3-A just cleaned.

The mechanism already exists and is proven: `tests/end-of-session/foundation.test.ts` (EOS-007) pins
one module's manifest and enforces barrel-only imports. F-044 calls it *"the repo's best idea,"* used
by exactly one module, while AJS-007 §6.1 lists Public-Surface Validation as a **validated
principle**. This task extends it per the REX-D8 ruling.

---

# Objective

Every surface M3-A deliberately settled is pinned by a manifest test that is **proven able to fail**,
so an accidental surface change in a later PR turns CI red.

---

# Scope

## Included (per the REX-D8 ruling — Option A recommended)
- A `foundation.test.ts`-style manifest for each module **REX-302 settled**: `context-builder`, and
  the `platform/*` + `products/knowledge-assistant` barrels whose `export *` was converted.
- Barrel-only-import enforcement extended to those modules (the second half of the EOS-007 pattern).

## Not Included
- **Modules M3-A did not touch** — Option B in REX-D8, ruled scope creep. Pinning an untouched
  surface freezes it before anyone decided what it should be. Record as post-REX; do not action.
- Any change to the surface itself — that was REX-301/REX-302. This task **pins**, it does not
  **decide**.

# Implementation Notes

**Pin the surface REX-302 chose, not the surface that exists.** The manifest asserts the deliberate
export list, cross-checked against the consumer inventory. A pin that simply snapshots
`Object.keys(module)` at authoring time pins accidents too.

**Every pin is proven able to fail — the M2 gate lesson, non-negotiable.** Add a stray `export` →
red; remove it → green. Record the demonstration. *A pin never seen red is not known to pin
anything*, exactly as M2's five gates each had to fail before they were trusted.

**Reuse the EOS-007 shape; do not invent a second pattern.** The value of F-044's idea is that it is
*one* idea. A divergent second manifest style would itself be an M3-B naming/consistency finding.

**This is test-only — no runtime behaviour.** Below the FPCP threshold; the scope guard is satisfied
because a manifest test changes nothing the platform *does*.
