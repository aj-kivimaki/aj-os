# Purpose

An export is a promise to a consumer. Today four barrels make that promise with `export *` (leaking
everything by default), one barrel re-exports effectively its whole module, one barrel's doc-comment
**claims a privacy it does not keep**, and one file is dead weight. None of these touches frozen
surface. This task makes each non-frozen barrel say — in code and in its doc-comment — exactly what
it means.

---

# Objective

Zero `export *` in `src/`; every non-frozen barrel pruned to what its consumers actually import; the
`context-builder` privacy claim **either true or deleted**; the dead file gone.

---

# Implementation Notes

**Name what consumers import, then stop.** For each `export *`, list the symbols the measured
consumers actually use and re-export exactly those. An export nothing imports is removed **or**, if
kept deliberately, gets a one-line reason — the DoD's *"removed or justified in writing."*

**F-038 is a judgement, and the pins are the floor.** Keep every symbol in `EXPECTED_OPERATIONS` and
every type a consumer imports; drop the rest. Do **not** prune below what a consumer needs to satisfy
a symbol count — the count is the trap M1 named, not the target.

**F-039 has two honest fixes, and the finding demands one of them:** either make the engines private
(if nothing outside the module needs them — check the 20 consumers first) or delete the false
sentence. **Do not leave a doc-comment that describes a surface the code doesn't have.**

**Every change is a contract decision, so the diff is reviewed export-by-export.** No rename may ride
along; if one seems required, that symbol belongs to M3-B.

**Prove the barrel change is behaviour-neutral:** the full suite is the witness. A pruned symbol that
was actually load-bearing turns a consumer red — that is the safety net working, not a setback.
