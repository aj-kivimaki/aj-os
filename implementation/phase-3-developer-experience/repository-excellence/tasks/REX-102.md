# Purpose

agreed at the SPEC-003 planning review, with a self-declared deadline:

> *"…and **no later than the SPEC-003 implementation Freeze Review** (AJS-007 Documentation

the location is a ratified decision (EOS-D2). §8 does not say which of its two outputs is
authoritative though EOS-D4 settles it. The backlog exists precisely to prevent *"silent divergence
between the specification and the frozen implementation plan"*, and the divergence happened anyway.

# Lessons Learned

- **A deferred item can rot in a way that its own deadline cannot prevent.** This backlog had an
  owner, a deadline, a rationale, and a named enforcement mechanism (AJS-007 §7.4) — and still went
  unapplied through five milestone freezes. **The deadline was inside the document nobody re-opened.**
  A deadline is only real if something outside the deferred item checks it.

- **The instruction to "bump the change log" described a document that does not exist**, and it took
  reading all eight specs to be sure rather than assuming SPEC-003 was the exception. The backlog's
  author reasonably assumed a convention the repository never had. **Cheap to verify, and it changed
  the action from "add a section" to "follow the precedent."**

- **The spec was not merely incomplete — it was silent on what is authoritative.** §8 listed
  `Session Summary` and `Review Package` as "Primary" outputs without saying that neither is the
  canonical one. A reader implementing SPEC-004 against the old §8 would reasonably have parsed the
  markdown. **The specification's most consequential defect was an omission, not an error** — which
  is exactly the kind that documentation review catches and testing never does.
