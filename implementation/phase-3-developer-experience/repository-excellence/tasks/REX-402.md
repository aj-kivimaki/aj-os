> **Status:** ⬜ Planned (M4)

> **Implementation Package:** REX — Repository Excellence Review · **Milestone:** M4

# Purpose

`KnowledgeAssistant` (385 lines) hard-wires every dependency as a field initializer — `new
ConfigService()`, `new PromptRenderer()`, `new AIClient()`, `new HandbookService(...)`. **You cannot
construct it without a real filesystem and a real API key**, so it has **zero tests** while the two
modules *with* composition roots have exhaustive suites. This task brings it to the repository's own
DI-and-testability standard.

---

# Objective

`KnowledgeAssistant` accepts its dependencies (a `deps` parameter or a composition root), is
constructible in a test with fakes, and has a test suite — following `createKnowledgePipeline` /
`createEndOfSessionWorkflow`, the patterns the repository already proves.

---

# Scope

## Included
- Introduce dependency injection: a `deps` parameter (or a `createKnowledgeAssistant` composition
  root) so `config`, `promptRenderer`, `ai`, and the handbook service are **injected**, not
  `new`-ed in field initializers.
- Preserve the existing production wiring: `ask.ts` still constructs a fully-real assistant with no
  behavioural change.
- Add a test suite exercising the assistant with injected fakes.

## Not Included
- Any behavioural change to what the assistant **does** — this is a constructability refactor, proven
  by identical behaviour.
- Agent-layer / handbook tests — REX-403.
- Implementing `ContextBuilderConfig`'s reserved fields or any deferred capability (scope guard).

# Implementation Notes

**Copy the repository's own pattern; do not invent one.** `createKnowledgePipeline` and
`createEndOfSessionWorkflow` are the exemplars — a factory taking a `deps` object with sane
production defaults. Match their shape so the repository has **one** DI idiom, not two.

**Behaviour must be identical.** The refactor moves *where* dependencies come from, not *what the
assistant does*. `ask.ts`'s real-wired path produces the same output on the same input — the proof is
the existing behaviour, unchanged.

**Injected fakes, not mocks of the world.** The suite should construct the assistant with in-memory
fakes for config/handbook/AI and assert the assistant's own logic — the thing that was untestable
before.
