# Purpose

Two configuration systems coexist: `src/config/{agentEnv,appEnv}.ts` (Zod over dotenv, `HANDBOOK_PATH`)
and `platform/config/ConfigService.ts` (hand-rolled validation over `aj.config.json`, `handbook.path`).
Two handbook-path sources of truth, two validation styles. **They serve two transports** — and
**merging them is a redesign that the scope guard defers.** This task documents the boundary so the
split is intentional and legible, not accidental.

---

# Objective

The two config systems are documented — what each governs, why they are separate, and that merging is
deferred (likely to the MCP migration) — so a reader understands the split rather than tripping over
it.

---

# Scope

## Included
- Document the boundary (a short note where a developer meets config — `docs/guides/configuration.md`
  and/or the module READMEs): `src/config` is the **dotenv/env** layer for the agent + API transport;
  `platform/config` is the **`aj.config.json`** service for the CLI/platform.
- State explicitly that the split is **deliberate for now** and that unification is deferred (MCP
  migration), citing the scope guard.

## Not Included

- Any behavioural change.
