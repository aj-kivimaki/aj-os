# Purpose

**Why does this task exist?**

`CONTRIBUTING.md:83` states rules for pull requests. **Nothing surfaces them at the moment a pull
request is opened** — no template. Nine dependencies drift with **no monitoring and no security
alerts** — no `dependabot.yml`. No `CODEOWNERS`.

**This is governance *process*, distinct from REX-201's governance *enforcement*.** CI decides
whether code is *correct*; these decide whether the *process around it* is visible. The frozen plan
grouped both under `.github/` **by path**; they answer different questions and fail in different ways —
a broken CI blocks a merge, a missing template just quietly lets standards slip.

---

# Objective

Make the contribution process visible at the moment it applies.

## Not Included
- CI workflow — **REX-201**.
- `SECURITY.md`, `CODE_OF_CONDUCT.md` — **REX-206** (documentation).
- Branch protection / required checks — **a repository setting, not a file.** Recommend; REX cannot set it.
- Issue templates — **deferred**. No evidence of need on a single-maintainer repository; adding them
  is speculative. Record.

# Implementation Notes

**The template must not become a second CONTRIBUTING.** If it restates the rules, the two drift —
the exact defect M1 spent a milestone correcting (a hand-maintained copy of a status that lives
elsewhere). **Link, checkbox, stop.**

**Keep it short enough to be read.** A template nobody fills in honestly is worse than none: it
manufactures the appearance of a check.

**Dependabot: monthly, grouped.** Nine dependencies on a single-maintainer repository do not need
weekly noise. **The finding is "unmonitored", not "not updated often enough."**

**`CODEOWNERS` with one owner is nearly a no-op today** — and worth adding anyway: it makes review
ownership explicit rather than assumed, and costs one line.
