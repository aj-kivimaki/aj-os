# REX-502 — Errors Share a Taxonomy and a Voice

> **Status:** ⬜ Planned (M5)
>
> **Protected outcome:** **the repository's errors share a base, a message voice, and cause chaining — without changing behaviour**
>
> **Implementation Package:** REX — Repository Excellence Review · **Milestone:** M5
>
> **Findings:** F-060, F-061, F-062, F-063

---

# Purpose

**12 custom error classes across 11 files each `extends Error` directly**, with no shared base, no
codes, and **zero `cause` chaining**. `catch (e) { if (e instanceof AjError) }` is impossible, so every
call site enumerates concrete classes — and F-060's live reproduction showed the enumeration is already
incomplete (`SourceConnectorError` missing from `wiki.ts`'s friendly-message path). Messages disagree
on spelling and punctuation. This task gives the errors one taxonomy and one voice.

---

# Objective

A shared `AjError` base makes `instanceof AjError` possible; message spelling and terminal punctuation
are consistent; wrapped errors chain their `cause`. **No behavioural change** — every class keeps its
name and message intent; only the hierarchy and construction internals change.

---

# Scope

## Included
- **F-060** — introduce a shared base (`AjError extends Error`) that the 12 classes extend, enabling a
  single `instanceof` at catch sites. Backward-compatible: each concrete class still exists and still
  matches its own `instanceof`.
- **F-061** — one spelling for the NUL-byte message across the three stores/guards.
- **F-062** — consistent terminal punctuation across error messages.
- **F-063** — `cause` chaining where an underlying error is wrapped (the underlying error is currently
  discarded).

## Not Included
- **Changing which errors are thrown, or when** — behaviour is fixed.
- Error **codes** as a new public contract — that is API surface (defer unless the reviewer scopes it
  in); the base + `cause` is the craft improvement.
- Rewriting the friendly-message catch sites' *logic* beyond adopting the shared `instanceof` (the
  `wiki.ts` gap the shared base closes is a natural consequence, not a separate feature).

## Preserve
- **`session.ts:148`'s "`AIError` deliberately absent" comment (preserve-list)** — the base must not
  tempt a "catch-all" that reintroduces the case that comment documents as impossible.

# Implementation Notes

**Backward compatibility is the whole game.** `class ConfigError extends AjError extends Error` — every
existing `instanceof ConfigError` still holds, every message intent is preserved, and the new
`instanceof AjError` is additive. Prove it by the suite: no test that catches a specific error changes.

**Where the base pays off, adopt it minimally.** The friendly-message catch sites (`session.ts`,
`wiki.ts`) can narrow to `instanceof AjError` — which closes F-060's `SourceConnectorError` gap as a
side effect. **Do not** rewrite their surrounding logic, and **do not** touch the `AIError`-absent
comment's guarantee.

**`cause` is construction-internal.** Adding `{ cause }` to a wrap changes no control flow; it attaches
the underlying error for diagnostics. Add it where an error is caught and re-thrown as a domain error.
