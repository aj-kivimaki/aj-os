# Purpose

**Why does this task exist?**

**The configuration says things that are not true.**

- `implementation/CLAUDE.md` § Code Quality requires code to **"pass linting"**. **There is no
  linter.** The instruction is unenforceable (F-028).
- `tsconfig.json:26-32` **comments out six strictness flags** the project evidently values — it
  enables far stricter ones right above them (F-031).
- `tsconfig.json:38` sets **`jsx: "react-jsx"`** in a Node-only repository with **no JSX anywhere**
  (F-034).

Three findings, one theme: **config that declares what the project isn't, or declines to enforce
what it claims to value.** The frozen plan allocated them by path (`tsconfig.json`); the re-read
allocated them by outcome — they are one job.

---

# Objective

Make the configuration true: a linter that exists, the dormant flags enabled, the dead config gone.

---

# Scope

## Included
- Biome linter ([REX-D7](../decisions/REX-D7.md)) + `lint` script + CI gate.
- The six flags: `noImplicitReturns`, `noImplicitOverride`, `noUnusedLocals`, `noUnusedParameters`,
  `noFallthroughCasesInSwitch`, `noPropertyAccessFromIndexSignature`.
- Remove `jsx: "react-jsx"`.

## Not Included

- Fixing lint findings **beyond what the gate requires**. The scope guard defers the rest.
- **Removing the dead code M3-A owns** (F-041, F-042) — those need FPCPs and are M3-A's.

# Implementation Notes

**The one real decision is `ProviderMetadata`.** An unused **type** in a schema file. Is it dead, or
is it a contract the schema documents but does not reference? **Read it before deleting it.** If it
is genuinely dead, removing it is a `src/` change — legitimate under
[REX-D10](../decisions/REX-D10.md) **only because a diagnostic named it**. If it is intentional,
that is a finding for M3-A, not a deletion here. **Do not guess; the file is small.**

**Enable the flags one at a time** and record what each produces. Six at once gives a pile of errors
and no information about which flag is worth having.

**Every `src/` change must be diagnostics-driven.** REX-D10's clause. If the linter suggests
something no diagnostic requires, it is out of scope — record it, do not do it.

**Suppress narrowly or not at all.** A baseline-suppress must carry a comment naming the milestone
that removes it. **Never suppress silently** — that converts a finding into a lie.
