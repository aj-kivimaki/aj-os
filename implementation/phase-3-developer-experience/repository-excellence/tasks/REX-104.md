# Purpose

**Why does this task exist?**

`docs/guides/` is where README sends anyone who wants to actually run AJ-OS, and it **actively
misleads them**.

`installation.md:51-56` tells a user that the wiki generator is *"implemented but **not yet wired to
a runnable command**"* and that seeing *"no generated wiki"* is *"a **known limitation**, not a setup
mistake."* `README.md:58` documents `aj wiki build`. **The two documents directly contradict each
other**, and the guide's advice is to accept a broken state that a shipped command fixes.

`configuration.md` calls itself *"Reference for AJ-OS settings"* and omits two of the three shipped
`handbook` keys — including `generatedWikiPath`, which README:68 calls *"one configuration
contract"*, and `reviewPath`, without which `aj session end` cannot be configured. It documents no
command beyond `aj ask`.

---

# Objective

Make the guides describe the commands and configuration that actually ship, so that following them
produces a working system.

# Implementation Notes

**Document configuration from `ConfigService.ts`, never from memory or the decision records.** The
service is what runs. Its defaults are the defaults. If `aj.config.example.json` and `ConfigService`
ever disagree, **halt and report (§3)** — that is a defect, not a documentation choice.

**F-002 is the highest-value fix in this task.** It doesn't merely omit a feature; it tells the user
that a broken state is expected and instructs them to tolerate it. A user following this guide today
concludes AJ-OS is half-built. Remove the limitation text, remove the Troubleshooting entry, and
document `aj wiki build` as the answer.

**Cross-check flags against `src/cli/index.ts` character by character.** A guide with a wrong flag is
worse than a guide with no flag: the first fails confusingly, the second sends the reader to `--help`.

**`aj session end` needs a home, not a mention.** It is the newest shipped capability and is
documented nowhere a user will look. It writes to `<vault>/knowledge-review/pending/<session-id>/`,
takes `--since <ref>` and `--notes <text>`, and **never touches canonical knowledge** — a property
worth stating, since it is the thing a cautious user will want to know before running it against
their vault.

**Scope guard check.** Every edit documents an existing implementation. Nothing changes behaviour.
**In scope, cleanly.**
