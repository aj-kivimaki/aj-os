# Implementation Notes

**Rename with `git mv`, one file (and its importers) per commit** — so `git show --stat` shows a
rename and the content diff is empty but for import specifiers. **Do not** hand-delete-and-recreate;
that loses the rename and its history.

**Restore-by-rewrite, never `git checkout`, on uncommitted work** (the REX-303 lesson) — M3-B does
many renames; a `git checkout <file>` to undo a probe reverts uncommitted rename work.

**Prove surface-neutrality per rename:** the REX-303 manifests and `foundation.test.ts` stay green,
and `git diff` on the renamed file shows only the import-path line. If an export identifier moves, the
manifest goes red — stop, because that is M3-A's frozen territory.

**The 4-file rename is import-heavy.** Measure importers first (`grep -rl` the old specifier), rename,
update, typecheck. The factories are wired in composition roots and tested widely.
