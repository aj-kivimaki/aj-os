# Purpose

**Why does this task exist?**

Because **AJS-007 §7.4 Documentation Synchronization is Mandatory, and it failed silently at two
separate reviewer-declared freezes.**

`src/end-of-session/README.md:5,7,54` still states *"Status: Milestone M1 — Foundation & Contracts
complete… **No behavior exists yet** — no git access, LLM calls, persistence, projection rendering,
or CLI."* All five milestones are frozen and `aj session end` ships. `src/context-builder/README.md`
still says M4 is *"pending Freeze Review"* though SPEC-002 M4 is frozen and v2.0.0 shipped Assembly.

`SPEC-FREEZE-REVIEW.md` Step 6 explicitly requires *"Module README updated (status line +
milestone/focus table)"* before a freeze — and `MILESTONES.md` v1.32 claims *"Documentation
synchronized per AJS-007 §7.4"*. **The claim was made and the check did not catch it.**

That is not a documentation defect. It is evidence that **a Mandatory mechanism has no verification
step**, and it belongs in REX-101's retrospective as much as it belongs here.

# Implementation Notes

**Status comes from MILESTONES, never from the README being edited.** Each package's `MILESTONES.md`
is the roadmap of record — `SPEC-FREEZE-REVIEW.md:142` says so. The READMEs drifted precisely because
they carry a hand-maintained copy of a status that lives elsewhere.

**On F-011 — remove the count, do not update it.** *"Current size: 205 tests across 15 files"* is
wrong today (207) and any number written there will be wrong again. The word *Current* is what makes
it a live claim. **The count is the defect, not its value.** Replace it with the guarantee the
sentence is actually trying to convey — the suite is deterministic and fast — which is what the
surrounding lines already say and which cannot drift.

**On F-024.** `SPEC-FREEZE-REVIEW.md:142` likely means *"there is no separate ROADMAP per
implementation package"*, which is true, but it reads as a flat contradiction of a root `ROADMAP.md`
that exists. **Clarify the scope; do not delete the sentence** — it is making a real point about
where a package's status of record lives.

**On F-020, sequencing matters.** The README advertises `retrospectives/ (added at each Milestone
Freeze)`. **REX-101 creates it.** If REX-101 is deferred or its shape changes, this line must
describe what actually exists — not what was planned. Do not fix F-020 by deleting the line; fix it
by making the line true.

**Do not rewrite. Correct.** These READMEs are well written and their authors were not careless — the
freeze checklist that was supposed to catch this did not. Change the status lines and the "arrives
later" tables. Leave the prose.

**Scope guard check.** Every edit corrects a false status claim about existing code. **In scope,
cleanly.**
