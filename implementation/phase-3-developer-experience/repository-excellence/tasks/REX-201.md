# Purpose

**Why does this task exist?**

**`.github/` does not exist and never has** — `git log --all -- .github` returns nothing across the
repository's entire history. Eleven pull requests merged with **zero automated gates**.

Meanwhile the repository states the policy **twice**: `CONTRIBUTING.md:72` and
`docs/guides/development.md:51` both require `npm run typecheck`, `npm run build`, and `npm test`
before every pull request. **A policy stated twice and enforced zero times.**

Every other M2 task builds a gate. **This task is the thing that runs them.** Without it, M2 produces
configuration nobody executes — which is the failure mode that produced this inventory.

---

# Objective

CI runs typecheck, build, and test on every push and pull request, and is **demonstrated failing**
before it is trusted.

# Implementation Notes

**Keep it boring.** `actions/checkout` + `actions/setup-node` + `npm ci` + the three gates. No
matrix, no caching cleverness, no reusable workflows. The finding is *"nothing runs"*, not *"what runs
is unsophisticated"*.

**The pipeline must be red before it is trusted.** Push a deliberate type error, confirm CI fails,
revert. **A gate never seen red is not known to work** — SPEC-003's standard, and the reason its
canonical-unchanged proof was tampered with before being believed.

**15 test suites touch `node:fs` / `node:child_process`** and construct real git repositories.
`session-fixtures.ts` runs `git init` and commits. **CI has no git identity configured by default**,
so those suites may fail in CI while passing locally. **Verify by running CI on the branch before
requiring the check** — this is the most likely source of a red pipeline that is CI's fault rather
than the code's.

**`npm ci` requires `package-lock.json`.** Confirm it is committed; if not, that is a finding, not a
workaround.

# Lessons Learned

- **The first gate is the cheapest one to get wrong and the most expensive one to distrust.** All
  three were proven able to fail before being believed — which took minutes and is the only reason
  the pipeline means anything. SPEC-003's canonical-unchanged proof set this standard; it costs
  almost nothing to apply.

- **Two of this task's three planned risks did not exist.** The git-identity concern was already
  handled by the fixtures' authors. **Planning correctly identified where to look; the code had
  already been there.** Worth noting for the M2 retrospective: a risk register's value is partly in
  the risks it retires.

- **CI now runs but cannot block anything.** Branch protection is a repository setting, so the
  finding *"eleven PRs merged with zero gates"* is only **half** closed by this task — the gate
  exists; requiring it is the reviewer's. **A gate that can be merged past is a suggestion**, and
  recording that honestly matters more than claiming F-025 is fully resolved.
