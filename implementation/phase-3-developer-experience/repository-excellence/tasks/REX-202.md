# Purpose

**Why does this task exist?**

**`npm run typecheck` type-checks zero test files.** Verified: `tsc --noEmit --listFiles | grep -c '/tests/'` → **0**.

`tsconfig.json:46` sets `include: ["src"]`, and `typecheck` runs `tsc --noEmit` with no `-p`, so that
file governs. Roughly **11,000 lines of test code** — a suite the same size as the source it
validates — have **never been type-checked**.

**This is not hypothetical.** The gap hid **three real defects during SPEC-003** (EOS-401's broken
`GitPort` stubs, EOS-405's `exactOptionalPropertyTypes` error, EOS-407's import of a type the barrel
does not export). Each was invisible to **both** checks: Vitest strips types, and the compiler never
looked.

**The `include: ["src"]` decision was correct — for `build`.** `tsconfig.json:38-41` explains it:
keeping tests out of `rootDir` keeps `npm run build` green. That reasoning is sound and this task
does not overturn it. **It was extended to `typecheck` for free, and `--noEmit` has no `rootDir`
problem to solve.**

---

# Objective

Make the type-checker see `tests/`. **Make the errors visible; do not fix them** — that is REX-203.

# Implementation Notes

**⚠️ `rootDir` must be widened, and this is the task's one real trap.** Verified 2026-07-17: a
`tsconfig.test.json` that merely extends `tsconfig.json` and adds `include: ["src","tests"]`
inherits `rootDir: ./src` and produces **58× TS6059** (*"not under rootDir"*) — one per test file.
**It never reaches typechecking at all**, and would report a wall of errors that look real and are
not. Set `rootDir: "."` (or drop it — `noEmit` makes it moot).

**The frozen plan did not mention this**, and a naive implementation would have produced 58 phantom
errors and a false picture of the work.

**Do not widen `tsconfig.json` itself.** `build` emits to `dist/` and `rootDir: ./src` is what keeps
`dist/` free of `tests/`. That constraint is real and documented at `tsconfig.json:38-41`. **Two
configs, two jobs** — the split the original decision reasoned toward but only half-applied.

**`typecheck` should run both** — `tsc --noEmit && tsc -p tsconfig.test.json`. The first keeps
`build`'s exact contract honest; the second covers everything. **Do not replace the first with the
second**: they check different programs, and only the first matches what ships.

# Lessons Learned

- **The `rootDir` constraint was the whole task, and planning found it.** A `tsconfig.test.json` that
  merely extends and adds `include` inherits `rootDir: ./src` and produces **58× TS6059** — never
  reaching typechecking at all. **The first implementation would have reported a wall of errors that
  look real and are not**, and the false picture would have shaped REX-203's estimate. The re-read
  paid for itself here, before a line was written.

- **A correct decision was extended one step too far, and that step cost months.** `include: ["src"]`
  is right for `build` — `rootDir` keeps `dist/` clean, and `tsconfig.json:38-41` says so. It was
  inherited by `typecheck` for free, where `--noEmit` has no `rootDir` problem to solve. **Ask what a
  setting is for before inheriting it.**
