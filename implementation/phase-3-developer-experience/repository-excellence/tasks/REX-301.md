# Purpose

Three symbols on the **frozen** public surface are declared but inert (F-041, F-042, F-043). Each is
frozen by a ratified decision — CB-002, ADR-006, SPEC-005 respectively — so the fix cannot be chosen
by the author. This task **executes only the dispositions [REX-D5](../decisions/REX-D5.md) rules**,
each ruled **before** the change it authorises (§7.2).

**This task may legitimately change zero lines of code.** If REX-D5 rules *document* on all three,
the frozen surface is untouched and the doc-comments are written in REX-302. That is a **result, not
a failure** — the REX-D3 principle applied to dead code.

---

# Objective

Every frozen-surface finding is closed **through** its FPCP, not around it: removed where a ruled
FPCP authorises removal, or documented as deliberately reserved/staged/declared where it does not.

# Implementation Notes

**Re-measure before you touch.** Confirm each symbol is still inert at implementation time with its
REX-D5 command — the inventory's line numbers are pre-formatter and must not be transcribed.

**A removal must be proven able to fail loudly first.** Before removing any frozen symbol, confirm a
consumer *would* break if it were load-bearing — i.e. the removal is safe *because* the symbol is
inert, demonstrated, not assumed. For F-042 specifically, a removal deletes two passing test suites;
that is a signal to re-check the ruling, not a step to perform quietly.

**Cite the authority in the artefact, not just the commit.** Every closure names CB-002 / ADR-006 /
SPEC-005 — the record that made the symbol frozen — so a future reader sees *why* it was
reserved/staged/declared, not merely that it was.
