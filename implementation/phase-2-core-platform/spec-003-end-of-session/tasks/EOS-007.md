# Purpose

Establish the permanent test foundation for the End-of-Session contracts.

> **Why does this task exist?** M1 delivers contracts and seams; their guarantees
> (validation, immutability, deterministic registry ordering, public-surface-only
> access) must be protected by a permanent test suite before behavior milestones
> build on them (AJS-007 Public-Surface Validation, Working Increment).

---

# Objective

Author permanent contract/boundary tests for every M1 deliverable, importing
**only** through the module's public surface (`src/end-of-session/index.js` /
`contracts` barrel), never internal files.

---

# Scope

## Included

- Tests for `SessionContext`, `Session` (EOS-002)
- Tests for `CandidateKnowledge` (EOS-003) — valid/invalid/immutable + governance-
  state pinning + provenance sanity
- Tests for `ReviewPackage`, `SessionReport` (EOS-004)
- Tests for `SessionChange`/`ChangeSet` + analyzer registry boundary (EOS-005) —
  including "held, not executed" via a throwing fixture analyzer
- Tests for `ManualTriggerSource` / `NoopNotificationPort` (EOS-006)
- A minimal shared local factory pattern per the existing test convention (inline
  factories, no shared helper library)

## Not Included

- Behavior tests for collection, extraction, generation, persistence, CLI (each
  behavior milestone owns its own permanent suite)
- Any network or real-git access

# Implementation Notes

- Follow the established test patterns: import from `index` barrels only; inline
  local factories (`makeSession()`, `makeCandidate()`, …); no reaching into
  internal modules (the SPEC-002 rule).
- For the registry, use a **throwing fixture analyzer** to prove "held, not
  executed" (CB-007 precedent).
- Assert deep immutability (mutation attempts throw) on each contract, mirroring
  the CB-003 package tests.
- Keep tests deterministic; no clock/network. Where a timestamp is needed, pass a
  fixed ISO string.
- Record the resulting test count in the task Worklog and the MILESTONES change log
  (the SPEC-002 convention of tracking suite growth).
