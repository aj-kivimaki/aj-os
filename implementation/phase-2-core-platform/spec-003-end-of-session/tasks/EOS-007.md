# Purpose

Establish the permanent test foundation for the End-of-Session contracts.

> **Why does this task exist?** M1 delivers contracts and seams; their guarantees
> (validation, immutability, deterministic registry ordering, public-surface-only
> access) must be protected by a permanent test suite before behavior milestones
> build on them (AJS-007 Public-Surface Validation, Working Increment).

---

# Objective

Author permanent contract/boundary tests for every M1 deliverable, importing
**only** through the module's public surface (`src/end-of-session/index.js` /
`contracts` barrel), never internal files.

# Implementation Notes

- Follow the established test patterns: import from `index` barrels only; inline
  local factories (`makeSession()`, `makeCandidate()`, …); no reaching into
  internal modules (the SPEC-002 rule).
- For the registry, use a **throwing fixture analyzer** to prove "held, not
  executed" (CB-007 precedent).
- Assert deep immutability (mutation attempts throw) on each contract, mirroring
  the CB-003 package tests.
- Keep tests deterministic; no clock/network. Where a timestamp is needed, pass a
  fixed ISO string.
- Record the resulting test count in the task Worklog and the MILESTONES change log
  (the SPEC-002 convention of tracking suite growth).

# Worklog Summary

Consolidated the module's test suite into a coherent foundation rather than adding
duplicate coverage — the M1 contracts and seams were already tested through the
public surface by EOS-002..EOS-006.

- **`tests/end-of-session/support.ts`** — one shared cross-cutting inspector,
  `firstUnfrozenPath(value)`: returns the dotted path of the first non-frozen nested
  object/array, or `null` when deeply immutable. A **pure inspector** (no internal
  assertion), so each call site keeps a visible `expect(firstUnfrozenPath(x))
  .toBeNull()` — the assertion stays where tooling/readers expect it (avoids the
  SonarLint S2699 "hidden assertion" warning) and a failure names the exact path.
- **Refactored** the five contract suites (`session`, `candidate`, `review-package`,
  `session-report`, `change`) to replace their ad-hoc, per-field `Object.isFrozen`
  blocks with the shared inspector. This eliminated the duplicated idiom (−48 lines)
  and *strengthened* the check (recursive over all nested values, not a hand-listed
  subset).
- **`tests/end-of-session/foundation.test.ts`** — new *module-wide* guards no single
  contract suite can provide: (1) the public **operations manifest** — exactly the
  intended eleven `parse*`/`create*` exports (drift guard for a minimal, intentional
  API); (2) **re-export completeness** — every contracts-barrel export is reachable
  from the module entry point; (3) a **public-surface-only import guard** — reads the
  suite sources and fails if any imports the module through a non-barrel path
  (permanently enforces AJS-007 Public-Surface Validation).

Design decisions:

- **Consolidated one inspector, not a factory library.** The task's convention
  (inline fixtures, no shared helper library) is preserved — each suite keeps its own
  `valid*` fixtures; only the single duplicated immutability idiom is shared, as the
  reviewer directed ("consolidate common helpers where appropriate").
- **Inspector, not assertion helper.** Returning the offending path (asserted
  externally) keeps assertions visible to tooling and yields better diagnostics than
  a helper that hides `expect` internally.

**Test count.** Full suite **438 tests / 42 files** (module suite: **98 tests / 10
files**). Net change from EOS-006 (435/41): +3 (the foundation suite), with the
contract-immutability tests consolidated rather than multiplied — coverage
strengthened, not merely increased.

**Validation.** `npm run typecheck`, `npm run build`, `npm test` all green. No test
imports a non-barrel path (enforced by the new guard). Reviewed against all 8
code-review angles — no correctness defects; two non-blocking observations
(regex-based import guard heuristic; intentional manifest-update friction) recorded.

# Lessons Learned

A shared *inspector* (returns data, asserted at the call site) beats a shared
*assertion helper* (hides `expect`): it keeps the assertion visible to linters and
readers, preserves per-test diagnostics, and still removes duplication — the clean
way to satisfy "consolidate helpers" without the over-abstraction the reviewer
cautioned against. The highest-value foundation tests are the ones no single
contract suite can express: the public-API manifest and the public-surface-only
import guard turn AJS-007's Public-Surface Validation from a convention into an
enforced, permanent invariant.
