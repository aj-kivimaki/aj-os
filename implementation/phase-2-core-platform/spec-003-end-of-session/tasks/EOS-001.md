# Purpose

Create the home for the End-of-Session Workflow.

> **Why does this task exist?** Every later task needs a module to live in and a
> public surface to export from. This task establishes that structure with no
> behavior, so subsequent contract tasks have a stable place to add to.

---

# Objective

Create `src/end-of-session/` as a new platform module with a public entry point
(`index.ts`) and an internal `contracts/` barrel. No contracts, services, or
behavior are implemented — only the module skeleton and its public export surface.

---

# Scope

## Included

- `src/end-of-session/index.ts` (public barrel; initially empty exports + header)
- `src/end-of-session/contracts/index.ts` (contracts barrel; populated by EOS-002+)
- A module README or header comment describing the module's responsibility and its
  single public entry point (`run(context)` — implemented later in M5)
- `tests/end-of-session/` test directory placeholder aligned to the module

## Not Included

- Any contract definition (EOS-002..EOS-005)
- Any service, analyzer, store, or CLI wiring
- The composition root or `run()` (M5)

# Implementation Notes

- Mirror the existing module conventions: a thin `index.ts` barrel, ESM `.js`
  import specifiers, no default exports.
- Keep the barrel honest — export nothing that does not yet exist. The header
  comment should state the single public entry point the module will expose
  (`run(context)`), so intent is clear before behavior lands.
- Follow the `contracts/` sub-barrel pattern chosen in EOS-D1 so consumers
  (SPEC-004) can import contracts without pulling in services.
