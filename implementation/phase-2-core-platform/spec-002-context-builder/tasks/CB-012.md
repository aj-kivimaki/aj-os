# Purpose

Extend the Context Builder's permanent automated test suite to validate deterministic knowledge collection.

This task verifies the collection behaviour implemented during Milestone 2.

The contract testing foundation established during CB-006 is reused.

No new testing framework is introduced.

---

# Objective

Create permanent automated tests for the knowledge collection pipeline.

The implementation should provide:

- Collection Engine tests
- Provider execution tests
- CollectionResult tests
- Context Builder integration tests
- deterministic behaviour tests
- documentation

The goal is to permanently protect the behaviour introduced during Milestone 2.

# Design Principles

Tests should be:

### Fast

Execute in milliseconds.

---

### Deterministic

No filesystem.

No network.

No randomness.

---

### Public API Oriented

Tests should validate only public behaviour.

---

### Readable

Tests should document expected platform behaviour.

---

### Stable

Implementation may evolve.

Public behaviour should not.

# Test Coverage

The automated tests should validate:

## Collection Engine

- public API
- immutable service
- deterministic behaviour

---

## Provider Execution

- deterministic execution order (registry order authoritative)
- provider invocation
- KnowledgeItem aggregation
- partial collection: a failing provider yields a CollectionError without aborting collection
- deterministic error ordering (provider completion order never influences result ordering)

---

## CollectionError

- runtime validation
- invalid CollectionErrors rejected
- immutability
- public contract

---

## CollectionResult

- runtime validation
- immutability
- public contract
- result containing both KnowledgeItems and CollectionErrors

---

## Context Builder

- collection pipeline
- service composition
- deterministic output
- integration of both successful knowledge and collection errors

---
