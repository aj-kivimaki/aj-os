# CB-022 — Deterministic Assembly

**Status:** Planned

> Milestone 4 — Context Assembly. Inherits the frozen M4 Planning Package and the
> frozen architecture (AD-001–AD-010). This document plans the task only; it
> introduces no code. CB-022 is the first Assembly **behaviour** task: it realizes
> the CB-020 and CB-021 decisions.

---

# 1. Purpose

Implement `assemble` — the deterministic construction of an immutable
`ContextPackage` from its inputs, **through the frozen CB-003 contract**
(`parseContextPackage()`), applying the CB-020 section-composition strategy and the
CB-021 metadata composition, with structurally valid minimal explainability and
summary (AD-001, AD-002, AD-007).

---

# 2. Scope

- **References:** derive `references` from `selectedItems[].source`, de-duplicated
  by `source.id`, preserving first-appearance order.
- **Sections:** compose `sections` per CB-020 — knowledge-derived sections
  populated by the structural mapping (order-preserving), the four named sections
  (Objective, Success Criteria, Constraints, Open Questions) present but empty.
- **Metadata:** compose `metadata` per CB-021 (provenance fields + injected
  `generated_at` + single-sourced versions).
- **Explainability & summary:** construct minimal, structurally valid values
  (`explainability = { summary: "", entries: [] }`, `summary = ""`) — present, not
  computed (RC-2).
- **Construct-through-contract:** produce the package via `parseContextPackage()`
  so structural invariants (unique ids/kinds, referential integrity) and
  deep-freeze hold by construction (RC-1).

---

# 3. Inputs

- `SelectionResult` (CB-014) — ordered `selectedItems` and provenance `metadata`.
- Injected `generated_at` (explicit input; CB-021 / Decision B).
- Version constants (CB-021).

---

# 4. Outputs

A single immutable, deep-frozen `ContextPackage` (CB-003), returned by `assemble`.

# 6. Implementation Notes

- **Construct-through-contract only** — no ad-hoc package object; the sole
  constructor is `parseContextPackage()` (RC-1). This is intrinsic construction,
  not the deferred semantic validation (AD-008).
- References de-duplicated preserving first-appearance order; sections
  stable-partitioned preserving the canonical `selectedItems` order (RC-6).
- Referential integrity by construction: every section `referenceId` resolves to a
  declared reference (CB-003 invariant).
- KnowledgeItems consumed unchanged — never reordered, filtered, merged, rewritten,
  or summarized (AD-002, Knowledge Identity).
- Pure transformation of explicit inputs — no ambient clock, no randomness, no
  external state; identical inputs → identical package (AD-007).
- Assembly does not render (AD-003) and does not compute explainability (AD-009).

---

# 7. Explicit Exclusions

- No `build(request)` integration (CB-023).
- No rendering (Markdown/JSON/MCP), semantic validation, explainability
  computation, token estimation, profiles, caching, or optimization (AD-009).
- No permanent behaviour tests — owned by CB-024.
- No modification of any frozen contract or of the frozen Planning Package.
