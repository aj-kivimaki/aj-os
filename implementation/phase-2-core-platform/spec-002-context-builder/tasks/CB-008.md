# Purpose

Define the immutable contract that represents a single knowledge collection failure.

The Context Builder uses a **partial collection with deterministic error reporting** model. Collection never fails because a single provider fails. A provider may contribute either KnowledgeItems or a CollectionError.

The CollectionError is therefore a first-class platform contract, defined **before** the CollectionResult so the result can embed collected errors alongside collected items.

This task defines only the error contract.

No collection behaviour is implemented.

---

# Objective

Create a stable, runtime-safe contract for collection errors.

The implementation should provide:

- CollectionError schema
- runtime validation
- immutable contract
- TypeScript types
- public exports
- documentation

The goal is to establish deterministic failure semantics for the Collection Engine.

---

# Failure Model

AJ-OS Context Builder collection is **partial**, not fail-fast.

- A single provider failure never aborts collection.
- A failing provider contributes a `CollectionError` instead of KnowledgeItems.
- Errors are collected deterministically and travel alongside items in the `CollectionResult` (CB-009).

# Design Principles

The Collection Error Model should be:

### Deterministic

The same failure should always produce the same error structure.

---

### Explicit

Errors are represented as platform contracts rather than implementation-specific exceptions.

---

### Immutable

Collection errors are immutable after creation.

---

### Provider-Agnostic

The model should not expose provider implementation details.

---

### Extensible

Future milestones may extend behaviour without changing the public contract.

# Public Contract

## CollectionError

Represents a deterministic platform description of a single collection failure.

The contract should contain only stable platform concepts.

Examples include:

- error identifier
- provider identifier
- failure category
- human-readable message

The exact schema should remain independent of provider implementation.
