# Purpose

Define the immutable contract representing the complete deterministic outcome of knowledge collection.

The CollectionResult becomes the canonical output of the Collection Engine and the input to future knowledge selection.

Because the Context Builder uses **partial collection with deterministic error reporting**, a CollectionResult represents the whole outcome and therefore contains **both**:

- collected KnowledgeItems
- collected CollectionErrors

This task defines only the contract.

No collection behaviour is implemented.

---

# Objective

Create a stable, runtime-safe contract for the output of the Collection Engine.

The implementation should provide:

- CollectionResult schema (items + errors + metadata)
- runtime validation
- immutable contract
- TypeScript types
- public exports
- documentation

The goal is to establish the platform contract for the complete outcome of collection.

# Design Principles

The CollectionResult should be:

### Immutable

Collection results are read-only after creation.

---

### Complete

The result represents the full deterministic outcome of collection — both items and errors.

---

### Deterministic

The same collected knowledge always produces the same CollectionResult structure.

---

### Minimal

Contain only stable platform concepts.

---

### Portable

Independent of implementation details.

---

### Extensible

Future milestones may enrich behaviour without changing this public contract.

# Public Contract

## CollectionResult

Represents the canonical, complete output of deterministic knowledge collection.

The contract should contain only stable platform concepts.

Examples include:

- collected KnowledgeItems
- collected CollectionErrors
- collection metadata

The exact schema should remain independent of execution strategy. An empty error collection is a valid result; a result containing both items and errors is a valid partial outcome.
