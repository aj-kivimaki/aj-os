# Lessons Learned

- **A test's import path is an architectural assertion.** Importing every test
  from the module's public entry point (`index.js`) â€” never an internal file â€”
  makes "test contracts, not implementation" a structural property, not a
  reviewer's judgement call. If a test needs an internal, that is a smell to
  investigate, not a path to lengthen.
- **`rootDir` and a test folder collide by default.** With `rootDir: ./src`, any
  `.ts` outside `src` fails the `tsc` build ("not under rootDir"). Scoping the
  compiler (`include: ["src"]`) and delegating test transform to the runner is
  the clean split â€” the build owns shipped code; the runner owns tests.
- **Deep-freeze deserves deep tests.** Asserting `Object.isFrozen` on nested
  arrays/objects (not just the root) is what actually protects the CB-003
  deep-freeze contract; a shallow check would pass even if the recursion broke.
- **Interfaces are tested by a fixture, not a schema.** `KnowledgeProvider` has
  behaviour, so a minimal in-test stub documents its shape and proves it is
  implementable â€” without adding any provider behaviour to `src`.
