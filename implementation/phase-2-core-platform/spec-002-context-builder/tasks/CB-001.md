No business logic is implemented during this task.

The implementation should establish a stable foundation for future milestones without implementing any Context Builder functionality.

# Implementation Notes

The module should:

- follow existing AJ-OS coding conventions
- expose a single public entry point (`index.ts`)
- keep internal components private
- favour composition over tight coupling
- avoid premature abstractions

Do not create placeholder classes or unnecessary files.

Only create structure that has a clear architectural purpose.

# Worklog Summary

**2026-07-07**

Established the Context Builder as an independent platform module at
`src/context-builder/`, mirroring the existing `src/agent/` module layout.

- Added `src/context-builder/index.ts` — the single public entry point,
  exporting a small `CONTEXT_BUILDER` identity constant (AJS-004 metadata seed).
  No business logic.
- Added `src/context-builder/README.md` documenting the module boundary,
  entry-point contract, and milestone roadmap.
- Added `tests/context-builder/README.md` as the test-folder placeholder; no
  test code, since CB-001 validation is limited to a successful build.

Validation: `npm run typecheck` and `npm run build` pass; the module imports
successfully and emits to `dist/context-builder/`. No circular dependencies.

**Decisions** (see `decisions/CB-001-scaffold-scope.md`):

- Minimal scaffold rather than pre-creating the full 9-folder tree. Subfolders
  are added by the tasks that introduce their code (CB-002+), honouring the
  task's "avoid premature abstractions / no unnecessary files" guidance.
- Test folder represented as a placeholder README only. A top-level `tests/`
  tree with `.ts` files would break `npm run build` because `tsconfig.json`
  sets `rootDir: ./src`, and no test runner is configured yet.

---

# Lessons Learned

- `tsconfig.json` uses `rootDir: ./src`, so future test tooling cannot rely on
  the default `tsc` build to compile a top-level `tests/` tree — the testing
  framework task (M1) must configure its own compilation/run path.
- The repository has no lint script and no test runner; "lint/tests" validation
  currently reduces to `typecheck` + `build`. Worth establishing a runner in
  the dedicated M1 foundation task before real tests are written.
