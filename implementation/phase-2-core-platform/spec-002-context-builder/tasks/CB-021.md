# 1. Purpose

Fix the complete deterministic **input set** to `assemble` and the **metadata
composition** rule for the `ContextPackage` (AD-006, AD-007). Builds on the
already-fixed section-composition model (CB-020).

---

# 2. Scope

- Define `assemble`'s complete explicit input set: the `SelectionResult` (CB-014)
  and the **injected `generated_at`** timestamp (Reviewer Decision B).
- Fix `ContextPackage.metadata` composition and field ownership:
  - `project`, `task`, `branch?`, `commit?` ŌĆö from `SelectionResult.metadata`
    provenance (the answered `KnowledgeRequest`; `branch`/`commit` optional).
  - `generated_at` ŌĆö a caller-supplied timestamp injected at construction time and
    passed to Assembly as an **explicit input** (no ambient clock).
  - `context_version` ŌĆö a fixed contract-version constant.
  - `context_builder_version` ŌĆö single-sourced from `CONTEXT_BUILDER.version`.
- Record that `KnowledgeRequest.issue` has no `ContextPackageMetadata` home and is
  intentionally dropped at assembly.
- Recorded as `decisions/CB-021-assembly-inputs-and-metadata.md`.

# 6. Implementation Notes

- **Determinism by construction (RC-3):** `generated_at` is an explicit input
  injected at Context Builder construction, so `assemble` remains a pure function
  of its inputs ŌĆö identical inputs (including the injected timestamp) yield an
  identical package. No ambient clock is read inside Assembly.
- Keeping `generated_at` an **explicit `assemble` input** preserves the CB-019
  boundary: `createAssemblyEngine()` stays construction-dependency-free; the
  timestamp is supplied through `assemble`, not held by the engine.
- Versions are single-sourced (closes the M1/M2/M3 deferred item): the two distinct
  constants are `context_builder_version ŌåÉ CONTEXT_BUILDER.version` and
  `context_version ŌåÉ` a fixed contract-version constant.
- Provenance metadata is reused, not recomputed (Compose Contracts) ŌĆö it flows from
  `SelectionResult.metadata` unchanged.
