# CB-019 ‚Äî Assembly Engine Service Boundary

**Status:** Done

> Milestone 4 ‚Äî Context Assembly. Inherits the frozen M4 Planning Package and the

> introduces no implementation and no code.

---

# 1. Purpose

Establish the **Assembly Engine** service boundary ‚Äî the platform service
responsible for constructing an immutable `ContextPackage` from a
`SelectionResult` (AD-001, AD-002).

This task establishes only the service boundary. It introduces no assembly
behaviour, no section composition, no metadata composition, no `ContextPackage`
construction, no `build()` integration, and no tests. It mirrors the Selection
Engine boundary (CB-013), which preceded `select` (CB-016).

---

# 2. Scope

- `createAssemblyEngine()` factory.
- Immutable `AssemblyEngine` service handle.
- The future stage operation `assemble(selectionResult, ‚Ä¶)` named in documentation
  only (its inputs are fixed by CB-021; its behaviour arrives in CB-022).
- Public exports through `src/context-builder/index.ts`.
- Module documentation.

The Assembly Engine holds nothing at construction ‚Äî like the Selection Engine, it
has no construction-time dependency; its inputs arrive at the `assemble` call.

---

# 3. Inputs

None at this task. The engine is composed during Context Builder construction and
accepts no runtime input until `assemble` is introduced (CB-022). The `assemble`
input set (a `SelectionResult` plus the injected assembly metadata) is fixed by
CB-020/CB-021, not here.

---

# 4. Outputs

An immutable `AssemblyEngine` service handle. No `ContextPackage` is produced.

# 6. Implementation Notes

- Follow the CB-007/CB-013 factory-service pattern: a co-located interface with
  its factory, **no `schema.ts`** (the Assembly Engine introduces no new data
  contract ‚Äî its output is the frozen CB-003 `ContextPackage`).
- The returned handle is `Object.freeze`-immutable and **stateless**.
- An empty handle is the honest minimal boundary: introduce **no** `assemble`
  stub and no placeholder members (CB-013 precedent). The stage operation is named
  in docs, not shipped as a stub.
- The Assembly Engine does not compose other pipeline stages and communicates only
  through immutable platform contracts (AD-002, Stage Independence).

---

# 7. Explicit Exclusions

- No `assemble` behaviour, section composition, metadata composition, or
  `ContextPackage` construction.
- No rendering, validation, explainability, profiles, or optimization (AD-009).
- No `build(request)` integration and no tests.
- No modification of any frozen M1‚ÄìM3 contract or of the frozen Planning Package.
