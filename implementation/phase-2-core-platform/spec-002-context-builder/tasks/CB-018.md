# CB-018 — Selection Behaviour Tests

**Status:** Planned

---

# Goal

Complete the permanent deterministic behaviour test suite for Milestone 3.

This task validates the Selection Engine, SelectionResult, Selection Policy, Context Builder pipeline extension, and deterministic selection behaviour introduced during Milestone 3.

This task introduces no production behaviour, no new contracts, and no architectural changes.

---

# Scope

This task includes:

- Selection Engine behaviour tests
- SelectionResult contract tests
- Selection Policy validation
- deterministic selection tests
- Context Builder pipeline tests
- end-to-end pipeline validation
- module documentation

This task extends the permanent Vitest foundation established in CB-006.

---

# Explicitly Excluded

Do not implement:

- Selection Engine features
- Selection Policy changes
- Context Builder behaviour
- Context Package generation
- explainability
- optimisation

Do not modify:

- SelectionResult
- CollectionResult
- Collection Engine
- Selection Engine
- Provider Registry

---

# Purpose

Validate the completed Milestone 3 implementation.

The test suite permanently verifies:

- deterministic selection
- canonical deterministic ordering (stable total order via the comparator chain)
- deterministic filtering
- duplicate elimination
- pipeline orchestration
- immutable contracts

The suite becomes part of the permanent platform regression tests.

---

# Test Coverage

## Selection Engine

Verify:

- immutable service handle
- stateless behaviour
- deterministic execution

---

## SelectionResult

Verify:

- valid contract
- invalid contract rejection
- deep immutability
- metadata preservation
- selectedItems preservation
- selectedItems ordering preservation (canonical order is the contract)
- no explicit priority field is exposed
- excludedItems preservation

---

## Selection Policy

Verify:

- deterministic evaluation
- deterministic filtering
- canonical deterministic ordering via the comparator chain
- stable total ordering guaranteed by the terminal immutable identifier tie-breaker
- exact duplicate elimination

---

## Selection Execution

Verify:

- identical CollectionResults produce identical SelectionResults
- KnowledgeItems are never modified
- SelectionResult is immutable
- policy execution is deterministic

---

## Context Builder Pipeline

Verify:

KnowledgeRequest

↓

Collection Engine

↓

CollectionResult

↓

Selection Engine

↓

SelectionResult

The Context Builder remains a thin orchestrator.

`build(request)` is the single public entry point.

The SelectionResult returned by `build(request)` matches direct `SelectionEngine.select(collectionResult)` execution, including identical canonical ordering of `selectedItems`.

---

## End-to-End Behaviour

Verify:

- repeated executions are identical
- deterministic pipeline ordering
- pipeline stage independence
- immutable pipeline contracts
- regression protection

---

# Design Principles

## Permanent Regression Suite

Tests remain permanently within the platform.

---

## Deterministic Validation

Tests verify deterministic behaviour rather than implementation details.

---

## Public Contract Validation

Tests validate only public contracts.

Internal implementation remains free to evolve.

---

## Pipeline Validation

Pipeline stages are validated independently and together.
