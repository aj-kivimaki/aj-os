# Architectural Goal

AJ-OS currently knows about business modules in multiple places.

Workspace Synchronization manually lists them.

Relations reference module keys.

Future dashboards, synchronization, CLI commands and analytics will also require knowledge of available modules.

Introduce a single Module Registry.

The Module Registry becomes the authoritative source for every registered business module.

---

# Your Role

You are a senior TypeScript engineer extending AJ-OS architecture.

Reuse the existing architecture.

Keep the implementation simple.

Avoid unnecessary abstractions.

---

# Objective

Implement a Module Registry.

Business modules should register themselves once.

Every other part of the application should consume the registry rather than maintaining its own module list.

The registry should contain metadata only.

It should not contain synchronization logic.

---

# Requirements

Create:

```
src/modules/

registry.ts
```

The registry should expose functions similar to:

- registerModule()
- getRegisteredModules()
- getModule()
- hasModule()

Business modules should register:

- Projects
- CRM

Each registration should include:

- module key
- display name
- database definition

Future metadata should be easy to add.

Example:

- icon
- version
- dependencies
- category

Do not implement future metadata yet.

---

# Workspace Synchronization

Update Workspace Synchronization.

It should no longer hardcode:

- Projects
- CRM

Instead:

Retrieve every registered module.

Synchronize each module in registry order.

---

# Relations

Relation validation should use the Module Registry.

A relation targeting an unknown module should produce a validation error.

Avoid string-based assumptions wherever possible.

---

# Constraints

Do NOT implement:

- Dependency ordering
- Plugin loading
- Dynamic imports
- Reflection
- File system scanning
- Dashboard integration

Keep the registry entirely static.

---

# Design Goals

The registry should be:

- strongly typed
- framework independent
- easy to extend
- the single source of truth

Avoid clever implementations.

Favor readability.

---

# Deliverables

Expected additions:

```
src/modules/

registry.ts
```

Updates to:

- Workspace Synchronization
- Relation validation
- Existing business modules

The application should function exactly as before.

The only behavioral change is replacing duplicated module knowledge with the Module Registry.
