# Architectural Goal

Introduce the Portfolio business module.

This milestone validates that AJ-OS can add new business modules with minimal changes to the surrounding architecture.

The Portfolio module should register itself through the Module Registry and participate in Workspace Synchronization automatically.

---

# Your Role

You are a senior TypeScript engineer extending the AJ-OS business layer.

Reuse the existing architecture.

Avoid unnecessary abstractions.

---

# Business Context

The Portfolio module represents publicly showcaseable work.

Each portfolio entry may originate from:

- a game project
- a freelance project
- a game jam
- a production music cue

For now, only Projects should be referenced.

---

# Objective

Create a complete Portfolio module following the same conventions as Projects and CRM.

The module should include:

- database definition
- strongly typed properties
- templates
- module registration
- relation definitions

No manual Workspace Synchronization changes should be necessary.
