# Objective

Implement the first business module of AJ-OS.

The Projects module represents the central entity of the freelance business.

It should describe projects using the existing Schema Engine and be completely independent of the Notion SDK.

No Notion API calls should be made.

---

# Business Context

Projects represent all meaningful work.

Examples include:

- Freelance game audio projects
- Personal portfolio projects
- Audio redesigns
- Technical audio demos
- Internal experiments

Every future module will eventually relate to Projects.

---

# Requirements

Create:

src/modules/projects/

Suggested structure:

```
src/modules/projects/

index.ts
database.ts
properties.ts
template.ts
```

Implement a reusable Projects database definition using the existing schema language.

Create a well-designed set of properties suitable for managing freelance game audio work.

Suggested properties include:

- Name (Title)
- Status
- Project Type
- Client
- Studio
- Start Date
- Target Completion
- Engine
- Middleware
- Repository
- Portfolio Ready
- Priority
- Notes

Use the existing:

- defineDatabase()
- defineProperty()

functions.

The module should export a complete Project database definition.
