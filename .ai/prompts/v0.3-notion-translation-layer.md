# Objective

Implement the translation layer that converts AJ-OS schema definitions into structures compatible with the Notion API.

This milestone should complete the separation between AJ-OS and the Notion SDK.

AJ-OS should continue speaking its own language.

The Notion layer should understand both AJ-OS and Notion.

---

# Requirements

Create a new translation layer inside:

src/notion/

Suggested structure:

```
src/notion/

translator.ts

properties/

    title.ts
    text.ts
    number.ts
    select.ts
```

Implement:

- NotionTranslator
- Property translators
- Database payload generation

The translator should accept:

DatabaseDefinition

and produce a structure representing a Notion database creation payload.

---

# Constraints

Do NOT create databases.

Do NOT call the Notion API.

Do NOT implement Projects.

Do NOT implement CRM.

Do NOT synchronize anything.

Do NOT add business logic.

Only implement translation.

---

# Design Goals

The translator should:

- Know about AJ-OS
- Know about Notion
- Hide all Notion SDK details from the rest of AJ-OS

The Schema layer must never import Notion types.

The translation layer may import both.

# Definition of Done

AJ-OS can translate a DatabaseDefinition into a complete Notion database payload without creating the database.

The schema layer remains completely independent of the Notion SDK.

The translation layer becomes the only place where Notion-specific implementation details exist.
