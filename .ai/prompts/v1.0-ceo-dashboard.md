# Product Goal

This milestone delivers the first complete AJ-OS experience.

The Dashboard is the primary entry point into the system.

It is **not** a database.

It is **not** a collection of database views.

It is an automatically generated executive report.

Its purpose is to answer one question:

> What does my business need from me today?

---

# Architectural Goal

Reuse the existing architecture.

Business Modules remain the source of truth.

Business Rules interpret the current business state.

The Dashboard Builder converts that information into a generated Dashboard page.

No business logic belongs inside dashboard widgets.

---

# Your Role

You are a senior software architect implementing the presentation layer of AJ-OS.

The architecture already exists.

Your responsibility is to present business information clearly.

Favor readability over cleverness.

# Today's Priorities

Generate a prioritized list of business actions.

Examples:

- Finish approaching deadline
- Contact overdue client
- Submit production music
- Publish portfolio item

Priorities should be generated automatically.

# Widgets

Each widget should expose a builder function.

Example

```
buildFinanceWidget()

buildProjectsWidget()

buildCRMWidget()
```

Widgets should receive business data.

They should not query Notion directly.

---

# Dashboard Builder

The builder collects every widget.

Produces one Dashboard model.

The Application Layer publishes that model to Notion.

Keep Dashboard generation independent from Notion APIs whenever possible.

---

# Notion Integration

Create a CEO Dashboard page.

The page should contain:

- headings
- summaries
- links to business databases

Simple formatting is sufficient.

Do not attempt advanced layouts.

Focus on information quality.
