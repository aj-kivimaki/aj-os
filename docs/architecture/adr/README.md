# Architecture Decision Records (ADR)

This directory contains the major architectural decisions made during the development of AJ-OS.

An Architecture Decision Record (ADR) documents:

- the problem being addressed,
- the decision that was made,
- the reasoning behind it,
- the consequences of the decision.

ADRs are intentionally lightweight.

They provide historical context and explain _why_ the architecture evolved the way it did.

---

## Guidelines

Create a new ADR whenever a decision:

- significantly changes the platform architecture,
- affects multiple services,
- introduces a new architectural principle,
- changes long-term direction,
- cannot easily be reversed.

ADRs should not document implementation details.

Those belong in the relevant SPEC documents.
