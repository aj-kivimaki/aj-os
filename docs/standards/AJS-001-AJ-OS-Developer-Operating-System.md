# AJ-OS Developer Operating System

## Core Philosophy

Every work session produces two deliverables:

-   A better project.
-   A smarter knowledge base.

Never let knowledge disappear inside commits.

Capture it.

Refine it.

Reuse it.

------------------------------------------------------------------------

# The Three AI Loops

## 1. Planning Loop (ChatGPT)

**Purpose**

Think. Not code.

### Responsibilities

-   Define objectives
-   Research
-   Compare alternatives
-   Design architecture
-   Break work into tasks
-   Produce implementation specifications

### Output

-   Implementation Specification
-   Success Criteria
-   Risks
-   Architecture Decisions

------------------------------------------------------------------------

## 2. Implementation Loop (Claude Code)

**Purpose**

Execute. Not brainstorm.

### Responsibilities

-   Read Context Package
-   Implement feature
-   Update tests
-   Update documentation
-   Stop

### Output

-   Working code
-   Updated project documentation

------------------------------------------------------------------------

## 3. Knowledge Loop (AJ-OS)

**Purpose**

Turn today's work into tomorrow's intelligence.

### Responsibilities

-   Extract
-   Filter
-   Organize
-   Review
-   Ingest

### Output

-   Updated handbook
-   Updated wiki
-   New playbooks
-   Better future context

------------------------------------------------------------------------

# Daily Workflow

## Phase 1 -- Define the Objective

### Checklist

-   [ ] One clear goal
-   [ ] Small enough for one coding session
-   [ ] Success criteria defined

------------------------------------------------------------------------

## Phase 2 -- Planning (ChatGPT)

### Checklist

-   [ ] Decide architecture
-   [ ] Research if necessary
-   [ ] Identify risks and edge cases
-   [ ] Produce implementation specification
-   [ ] Break into milestones

**Deliverable:** Implementation Specification

------------------------------------------------------------------------

## Phase 3 -- Build Context Package (AJ-OS)

AJ-OS gathers:

-   README
-   CLAUDE.md
-   Project documentation
-   Relevant handbook notes
-   Relevant wiki pages
-   Coding standards
-   Related source files

**Output**

`Context.md`

This should be the primary document given to Claude Code.

------------------------------------------------------------------------

## Phase 4 -- Implementation (Claude Code)

### Checklist

-   [ ] Read Context.md
-   [ ] Implement only the requested feature
-   [ ] Follow coding standards
-   [ ] Update tests
-   [ ] Update documentation
-   [ ] Stop when complete

### Rules

-   No planning
-   No architecture discussions
-   No unrelated refactoring

------------------------------------------------------------------------

## Phase 5 -- Review

### Checklist

-   [ ] Tests pass
-   [ ] Feature verified
-   [ ] Documentation updated
-   [ ] Code reviewed
-   [ ] Ready to commit

------------------------------------------------------------------------

## Phase 6 -- Commit

### Checklist

-   [ ] Clean commit
-   [ ] Meaningful commit message
-   [ ] Push changes

------------------------------------------------------------------------

## Phase 7 -- End-of-Session Automation

Automatically collect:

-   Git diff
-   Changed files
-   Commit message
-   Documentation updates
-   Architecture changes

Automatically generate:

-   Session summary
-   Decisions made
-   Lessons learned
-   Candidate handbook entries
-   Candidate wiki notes
-   Candidate playbooks
-   Candidate automation ideas

------------------------------------------------------------------------

## Phase 8 -- Knowledge Review

### Human approval required

-   [ ] Accept
-   [ ] Edit
-   [ ] Reject
-   [ ] Merge duplicates

Nothing enters the handbook automatically.

Nothing enters the wiki automatically.

Automation prepares.

You curate.

# Project Documentation

Purpose: Record project history.

Contains:

-   Experiments
-   Logs
-   Decisions
-   Research
-   TODOs
-   Architecture
-   Screenshots

------------------------------------------------------------------------

# Handbook

Purpose: Reusable knowledge.

Contains:

-   Playbooks
-   Checklists
-   Standards
-   Templates
-   Best practices
-   Lessons worth repeating

Rule:

> Will Future AJ use this again?

If yes, it belongs in the handbook.

------------------------------------------------------------------------

# Wiki

Purpose: Permanent AI memory.

Contains atomic knowledge only.

Examples:

-   OAuth
-   Fastify
-   MCP
-   n8n
-   Unity
-   Wwise
-   Git

No project history.

Only knowledge.

# The Golden Rules

1.  Think before coding.
2.  Build one feature at a time.
3.  Keep AI conversations focused.
4.  Start a new Claude session after every completed feature.
5.  Document while knowledge is fresh.
6.  Reuse knowledge instead of rewriting it.
7.  Automate repetitive work.
8.  Review all permanent knowledge.
9.  Every project should improve AJ-OS.
10. Every session should make Future AJ faster than Present AJ.
