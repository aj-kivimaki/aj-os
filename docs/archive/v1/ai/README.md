# AJ-OS v1 — AI Engineering Context (Archived)

> **Status: Archived (superseded).** This was the AI-collaboration context layer
> for AJ-OS v1 — the guidance an AI assistant read before implementing the
> code-first Notion business operating system. It does **not** describe how AJ-OS
> works today and is preserved, frozen, as a historical record. See
> [`docs/archive/v1/`](../README.md) and, for the current project,
> [`docs/VISION.md`](../../../VISION.md).

## Why this is archived rather than updated

This layer carries the v1 identity and v1 assumptions throughout: it frames
AJ-OS as a Notion business operating system, names specific tools of the time,
and predates the current Vision, Architecture, and Standards. Rather than evolve
these documents and risk carrying historical assumptions forward, the v2 AI
guidance will be **rebuilt from the current
[VISION](../../../VISION.md), [architecture](../../../architecture/), and
[specifications](../../../specifications/)** when it is needed — see the guiding
principle that the archive preserves history but does not keep historical designs
active.

## What is archived here

- `project-context.md`, `engineering-philosophy.md`, `coding-standards.md`,
  `implementation-rules.md`, `workflow.md` — the v1 AI operating guidance.
- [`prompts/`](prompts/README.md) — the v1 milestone prompt history
  (`aj-os-v1.0.0/`) and the early API foundation prompt (`api/`).
