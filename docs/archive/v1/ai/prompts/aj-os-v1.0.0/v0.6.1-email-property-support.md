# Your Role

You are a senior TypeScript engineer extending the existing AJ-OS architecture.

The framework, schema engine and translation layer already exist.

Your task is to complete support for one missing property type.

---

# Objective

The CRM module introduces an Email property.

The Schema layer already supports this property type, but the Notion Translation Layer does not.

Extend the existing translation system so Email properties can be translated into valid Notion database properties.

No other architectural changes are required.

---

# Current Situation

Supported property translators:

- title
- text
- number
- select
- multi_select
- date
- checkbox
- url

Missing:

- email

The CRM module cannot currently be translated because Email is unsupported.
